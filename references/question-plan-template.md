# Question Plan 模板

在正式生成 dev spec 前，先用这个模板整理阅读 / 分析 / 追问计划。

这个计划默认只在对话中展示，不默认落盘。

## 模板

````markdown

### 2. 待读取输入
- 

### 3. 当前不确定项
- 

### 4. 阻断项
- 

### 5. 待解决问题清单
1. 
2. 
3. 

### 6. 追问顺序与当前准备询问的问题
- 追问顺序：
  1.

  3. 
- 当前准备询问：

### 7. 是否继续追问的判断标准
- 若以下问题仍未关闭，则继续逐条追问：
  - 
  - 
- 若阻断项全部关闭，则进入正式文档生成
````

## 使用提醒

- 计划里只保留高价值主题，不要把所有阅读细节都堆出来
- 阻断项优先级高于普通不确定项
- 原生选择框可用时，按清单逐条询问
- 原生选择框不可用时，退回文本 `A/B/C/D` 兜底，每轮最多 3 个问题
