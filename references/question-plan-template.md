# Question Plan 模板

在正式生成 dev spec 前，先用这个模板整理阅读 / 分析 / 追问计划。

这个计划默认只在对话中展示，不默认落盘。

## 模板

````markdown
## 本轮计划

### 1. 已读取输入
- 

### 2. 待读取输入
- 

### 3. 当前不确定项
- 

### 4. 阻断项
- 

### 5. 本轮准备追问的 3 个问题主题
1. 
2. 
3. 

### 6. 下一轮是否继续追问的判断标准
- 若以下问题仍未关闭，则继续追问：
  - 
  - 
- 若阻断项全部关闭，则进入正式文档生成
````

## 使用提醒

- 计划里只保留高价值主题，不要把所有阅读细节都堆出来
- 阻断项优先级高于普通不确定项
- 3 个问题应尽量属于同一决策链路，避免一轮问得太散
