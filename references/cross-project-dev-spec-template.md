# Cross Project Dev Spec 模板

当一次需求涉及多个项目时，除各项目自己的 dev spec 外，还应额外生成这份跨项目总控文档。

这份文档的目标是把多项目的责任边界、协议一致性、开发顺序和联调顺序讲清楚。

## 写作提醒

- 不要把它写成项目级 spec 的简单拼接
- 重点是责任边界、依赖关系、协议一致性和顺序安排
- 若某个协议定义必须由单一项目主导，应明确写出“定义方”
